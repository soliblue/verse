# Morrow v0

## Goal

Ship a private iOS morning reader backed by a small VPS service. The app must open with a useful bundled edition, refresh prepared editions from the VPS, remain readable offline, collect feedback, and queue deep dives.

## Status

Complete locally. The first GitHub-hosted native iOS build is the remaining post-push gate.

## Contracts

- SQLite is the source of truth.
- `GET /v1/edition/today` returns one complete prepared `EditionPayload`.
- `GET /v1/editions` returns edition summaries and `GET /v1/editions/{id}` returns a complete payload.
- `GET /v1/topics` and `PUT /v1/topics` read and replace the editable topic set.
- `POST /v1/feedback` records saved, seen, more like this, less like this, and too basic signals.
- `POST /v1/deep-dives` queues a story for the next nightly run.
- Private API requests require `Authorization: Bearer <secret>` unless unauthenticated Tailscale mode is explicitly enabled.
- The app and server decode the same snake-case JSON schema.
- A successful nightly run stages and validates a new edition before replacing the current materialized payload.

## Decisions

- Use the Python standard library for the server and ETL so the VPS needs no application package install.
- Use SwiftUI, Observation, async/await, URLSession, and SwiftData with an iOS 17 deployment target.
- Bundle a prepared first edition in the app as the cold-start and offline fallback.
- Keep remote images optional and make the reading experience complete without them.
- Keep the v0 source collectors explicit and inspectable. Model enrichment is a replaceable local CLI boundary.
- Generate the Xcode project with XcodeGen on macOS, while checking in the generated project so cloning and building do not require XcodeGen.

## Verification

- Schema migration and seed are repeatable on an empty SQLite database.
- Server contract tests cover reads, writes, authorization, and persistence.
- ETL validation rejects malformed citations and incomplete editions.
- Swift model and fixture contract tests decode the bundled and server fixtures.
- GitHub Actions runs backend tests, a signing-free iOS simulator build, unit tests, and a simulator smoke launch.
- `make check` passes 55 tests plus Python compilation, shell syntax, and payload contracts.
- Tree-sitter parses all 67 Swift files and the Xcode project parser resolves all three native targets.
- Actionlint accepts the workflow, systemd accepts the rendered units, and the app icon and property lists validate.
- Repeatable bootstrap, preflight, SQLite integrity, and an isolated Codex protocol smoke pass on the VPS.
- A live deterministic Nightjar run collected 26 eligible source items and published a valid 10-item edition.
- Concurrent first-run migration stress passes 25 consecutive four-connection runs.
